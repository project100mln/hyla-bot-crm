# Hyla — Бот + CRM (готовый проект)

Один проект: Telegram-бот собирает и квалифицирует лидов, CRM показывает их
канбан-доской. Всё работает на одной общей базе данных — без Bitrix24 и
прочих платных CRM.

## Что внутри
- `bot.js` — Telegram-бот: квиз → сбор контакта → сохранение в базу
- `server.js` — веб-сервер: API для CRM + раздача интерфейса
- `db.js` — подключение к базе данных и автоматическое создание таблицы
- `public/` — сама CRM (HTML/CSS/JS, без сборки, открывается в браузере)

## Шаг 1. Создай бота в Telegram
1. Напиши **@BotFather** → команда `/newbot`
2. Придумай имя и юзернейм (например `hyla_almaty_bot`)
3. Скопируй токен, который он выдаст — это `BOT_TOKEN`

## Шаг 2. Узнай свой chat ID (куда слать уведомления о горячих лидах)
1. Напиши **@userinfobot** → `/start`
2. Он покажет твой `Id` — это `OWNER_CHAT_ID`

## Шаг 3. Деплой на Railway (бесплатно)
1. Зарегистрируйся на [railway.app](https://railway.app) (можно через GitHub)
2. Нажми **New Project → Deploy from GitHub repo**
   (если кода ещё нет на GitHub — можно загрузить эту папку как репозиторий
   через сайт github.com → "Create new repository" → "uploading an existing file")
3. После создания проекта: **+ New → Database → Add PostgreSQL**
   — Railway сам создаст переменную `DATABASE_URL` и подключит её к проекту
4. В настройках основного сервиса → **Variables** добавь:
   - `BOT_TOKEN` — токен из шага 1
   - `OWNER_CHAT_ID` — твой chat ID из шага 2
5. Railway сам выполнит `npm install` и `npm start` — через 1-2 минуты
   проект будет жить 24/7
6. Railway выдаст тебе публичную ссылку (Settings → Generate Domain) —
   это адрес твоей CRM. Открывай её в браузере с компьютера или телефона.

## Шаг 4. Проверка
- Напиши своему боту в Telegram `/start` — пройди квиз до конца
- Открой ссылку CRM — новый лид должен появиться в колонке "Новый лид"
  (CRM обновляется автоматически каждые 15 секунд)
- Если лид "горячий" (например указал аллергию + животных) — тебе придёт
  уведомление от бота в личные сообщения

## Если что-то не подключается
- Проверь, что `BOT_TOKEN` скопирован без пробелов
- Проверь, что Postgres в Railway добавлен именно к этому проекту
  (он должен появиться в списке сервисов рядом с основным)
- Логи запуска смотри в Railway → твой сервис → вкладка **Deployments → Logs**

## Локальный запуск (если хочешь протестировать на своём компьютере)
```bash
npm install
cp .env.example .env
# заполни .env своими значениями, для DATABASE_URL понадобится локальный
# или облачный Postgres (например бесплатный на supabase.com)
npm start
```
Открой http://localhost:3000
